MI (Mathematical Induction, 数学归纳法)

Axiom 2.5 (Principle of mathematical induction). Let P(n)P(n)P(n) be any property pertaining to a natural number nnn. Suppose that P(0)P(0)P(0) is true, and suppose that whenever P(n)P(n)P(n) is true, P(n++)P(n++)P(n++) is also true. Then P(n)P(n)P(n) is true for every natural number nnn.
-- Analysis I, by Terence Tao

数学归纳法通常分为两个关键步骤：

  1. 证明 “当n=1n=1n=1时命题成立”
  2. 证明 “若假设在n=mn=mn=m时命题成立，可推导出在n=m+1n=m+1n=m+1时命题成立，m∈Nm\in \mathbb{N}m∈N”

在第1步中，我们证明了起始情况(base case)
在第2步中，我们借助归纳假设(inductive hypothesis)，证明了命题

Reduction (归约)

Reduction is the single most common technique used in designing algorithms. Reducing one problem X to another problem Y means to write an algorithm for X that uses an algorithm for Y as a black box or subroutine. Crucially, the correctness of the resulting algorithm for X cannot depend in any way on how the algorithm for Y works. The only thing we can assume is that the black box solves Y correctly.
-- Algorithms, by Jeff Erickson

而在递归的上下文中，Reduction（“归约”），是将原问题 XXX 转化为一个或多个更简单的的同一类问题 X′(orX1′,X2′,⋯ ,Xn′)X' (or X'_1, X'_2, \cdots, X'_n)X′(orX1′​,X2′​,⋯,Xn′​) ，并认为这些简化后的问题可以被解决（从数学归纳法的角度，这里所谓“认为可以被解决的更简化的问题”，就是归纳假设！）；到这里，距离完成这个问题之差一步：base case（即数学归纳法中的“奠基步”、“初始情况”）

可以有如下直观：

  • 递归：通过假设递归调用的结果是正确的，从而解决当前问题。
  • 数学归纳：假设命题在规模较小的情况下成立，然后证明其在规模更大的情况下也成立。

另一个很有意义的直观：

When we design algorithms, we may not know exactly how the basic building blocks we use are implemented, or how our algorithms might be used as building blocks to solve even bigger problems. That ignorance is uncomfortable for many beginners, but it is both unavoidable and extremely useful. Even when you do know precisely how your components work, it is often extremely helpful to pretend that you don’t.
-- Algorithms, by Jeff Erickson

n!n!n!

阶乘定义如下：
n!=n×(n−1)×⋯×1,n∈N,n! = n \times (n-1) \times \cdots \times 1, \quad n \in \mathbb{N},n!=n×(n−1)×⋯×1,n∈N,
规定：
0!=1.0! = 1.0!=1.

同时，有其递归定义：
n!={1if n=0,n×(n−1)!if n≥1.n! = \begin{cases} 1 & \text{if } n = 0, \\ n \times (n-1)! & \text{if } n \geq 1. \end{cases}n!={1n×(n−1)!​if n=0,if n≥1.​

下面使用数学归纳法证明阶乘的递归定义：

(Base Case)
当 n=1n = 1n=1 时，由定义，1!=1=1×(1−1)!=1×0!=11! = 1 = 1 \times (1-1)! = 1 \times 0! = 11!=1=1×(1−1)!=1×0!=1，命题成立

(Inductive Hypothesis)
假设当 n=k,k≥2n = k, k \geq 2n=k,k≥2 时命题成立，即：
k!=k×(k−1)!=k×(k−1)×⋯×1k! = k \times (k-1)! = k \times (k-1) \times \cdots \times 1k!=k×(k−1)!=k×(k−1)×⋯×1

(Inductive Step)
当 n=k+1n = k+1n=k+1 时
(k+1)!=(k+1)×k×⋯×1=(k+1)×k!(k+1)! = (k+1) \times k \times \cdots \times 1 = (k+1) \times k!(k+1)!=(k+1)×k×⋯×1=(k+1)×k!

与阶乘的定义一致

可以根据阶乘的递归定义，自然地写出递归求 n!n!n! 的程序：

int factorial(int n) {
    if (n == 1 || n == 0) { // 实际上只需写 n == 0，但为了与数学归纳法表述对应
        return 1;
    } else {
        return n * factorial(n - 1);
    }
}

如何来看这样一个递归程序呢？函数factorial在“调用自己”时，会假设(或相信)factorial(n - 1)的返回值是(n−1)!(n-1)!(n−1)!，并且是正确的，然后再乘以n，就得到了n!n!n!——这时候我们会不自觉地想“factorial(n - 1)内部是怎么执行的？”——STOP!——一定程度上，这便是递归的“魔力”所在；但是这样的假设并非空想，其正确性是建立数学归纳法的基础上的：

  1. fractional(n-1)是我们的归纳假设——即我们已经（反向）完成了"inductive step"
  2. 我们已经对base case做了正确实现：if (n == 1 || n == 0) { return 1; }

因此，当我们试图思考factorial(n - 1)的内容时，从数学归纳法的角度来看，我们其实是从n=k−2n=k-2n=k−2，推导了n=k−1n=k-1n=k−1时成立，再从n=k−1n=k-1n=k−1推导了n=kn=kn=k时成立，显然，这是重复的工作，在数学归纳法中我们只需要做一步推导，同样的，在递归程序中，我们只需要做一次递归调用，而不需要再“递归地”关心递归调用

递归的实现：调用栈(Call Stack)

前面从数学归纳法的角度出发介绍递归，个人认为是很有趣也很有意义的一种方式——尤其是关于如何来读一个递归程序，但是，当涉及到递归内部的操作时，我们也需要明确其具体实现（从STOP!出发）

“递归”在计算机中的实现基于“调用栈(Call Stack)”——即函数调用的栈空间，“栈”是一种线性数据结构，具有“后进先出”的特性(LIFO, Last In First Out)，栈相关的内容可参考这里

不必深入讨论调用栈的细节，只需要建立直观：

  • 每次函数调用时，会将函数的参数、局部变量等信息压入栈中
  • 而当函数返回时，这些信息会被弹出栈

同样使用阶乘的递归程序来说明，使用printf展示递归调用栈时入栈与出栈的过程

入栈：

int factorial(int n) {
    if (n == 1 || n == 0) { // 实际上只需写 n == 0，但为了与数学归纳法表述对应
        printf("n = %d, 1\n", n, n);
        return 1;
    } else {
        printf("n = %d, factorial(%d - 1)\n", n, n);
        return n * factorial(n - 1);
    }
}

输出：（自上而下代表入栈顺序）

n = 4, factorial(4 - 1)
n = 3, factorial(3 - 1)
n = 2, factorial(2 - 1)
n = 1, 1

出栈：

int factorial(int n) {
    if (n == 1 || n == 0) { // 实际上只需写 n == 0，但为了与数学归纳法表述对应
        printf("n = %d, 1\n", n);
        return 1;
    } else {
        int output = n * factorial(n - 1);
        printf("n = %d, %d\n", n, output);
        return output;
    }
}

输出：（自上而下代表出栈顺序）

n = 2, 2
n = 3, 6
n = 4, 24

序列求和问题

递推:

int sum_list_iter(int *x, int size) {
    int sum = 0;
    for (int i = 0; i < size; i++) {
        sum += x[i];
        printf("%d ", sum);
    }
    return sum;
}

int main() {
    int x[] = {1, 2, 3, 4, 5};
    int size = sizeof(x) / sizeof(x[0]);
    int sum = sum_list_iter(x, size);
    printf("\nsum: %d\n", sum);
    return 0;
}

输出：

1 3 6 10 15 
sum: 15

递归:

int sum_list_recur(int *x, int size) {
    if (size == 0) {
        return 0;
    } else {
        int s = x[0] + sum_list_recur(x + 1, size - 1);
        printf("%d ", s);
        return s;
    }
}

int main() {
    int x[] = {1, 2, 3, 4, 5};
    int size = sizeof(x) / sizeof(x[0]);
    sum_list_recur(x, size);
    return 0;
}

输出：

5 9 12 14 15

我们会发现，递归的输出与迭代的计算过程不同，如果考虑递归调用栈，这一现象是显然的——上面程序中递归求和的时候加法是在递归的“出栈”后进行的

事实上，如果将加法执行的“位置”稍加调整：

int sum_list_recur(int *x, int size, int sum) {
    if (size == 0) {
        return sum;
    } else {
        printf("%d ", sum + x[0]);
        return sum_list_recur(x + 1, size - 1, sum + x[0]);
    }
}

输出：

1 3 6 10 15

修改后的sum_list_recur函数的参数中增加了一个sum，用于累计在入栈过程中进行的加法结果，从而得到与迭代相同的计算过程

虽然这里讨论求和的计算过程略显多余（主要是因为加法满足交换律，运算先后顺序并不影响结果），但是，当涉及到不满足交换律的操作时，明确递归的过程是很有意义的

斐波那契数列

递推:

递推地计算斐波那契数列：（每次计算仅依赖于前两项的信息，而前两项的信息已知）

int fib_iter(int n) {
    if (n == 0 || n == 1) {
        return n;
    }
    int prev1 = 0, prev2 = 1, curr;
    for (int i = 2; i <= n; i++) {
        curr = prev1 + prev2;
        prev1 = prev2;
        prev2 = curr;
    }
    return curr;
}

递归: 见递归树部分

参考前面的递归树，可以发现：fib_iter 只“遍历”了递归树的一条路径，而“递归”的做法则是遍历了整个递归树，因此前者的复杂度更优 (O(n)(O(n)(O(n) vs. O(2n))O(2^n))O(2n))

典例: Tower of Hanoi (汉诺塔问题)

  • 有三根柱子（A、B、C），在柱子A上有若干不同大小的圆盘，从上到下按大小顺序排列。
  • 目标是将所有圆盘从柱子A移到柱子C，并满足以下规则：
    1. 每次只能移动一个圆盘
    2. 大圆盘不能放在小圆盘上

reduction & base case

对于汉诺塔问题：

原问题：将 nnn 个圆盘从柱子 A 移动到柱子 C
归纳假设：将 n−1n-1n−1 个圆盘从柱子 A 移动到柱子 B(或C，由对称性)

将原问题归约到归纳假设，只需完成三步：

  1. 将 n−1n-1n−1 个圆盘从 A 移动到 B（借助 C）
  2. 将第 nnn 个圆盘从 A 移动到 C
  3. 将 n−1n-1n−1 个圆盘从 B 移动到 C（借助 A）

至此，问题已经基本解决！下面只需要考虑base case即可：对只有一个圆盘的情况，可直接移动

递归实现： 

void hanoi(int n, char from, char to, char aux) {
    if (n == 1) { // base case: 当只有一个圆盘时，直接移动
        printf("Move disk %d from %c to %c\n", n, from, to);
        return;
    }
    // divide/reduction: 将 n-1 个圆盘从 from 移到 aux，（借助 to）
    hanoi(n - 1, from, aux, to);
    // delegate: 将第 n 个圆盘从 from 移到 to, 这里的“操作”在形式上是移动圆盘
    printf("Move disk %d from %c to %c\n", n, from, to);
    // divide/reduction: 将 n-1 个圆盘从 aux 移到 to，（借助 from）
    hanoi(n - 1, aux, to, from);
}

时间复杂度

汉诺塔问题每次递归将 nnn 个圆盘分解为两个规模为 n−1n-1n−1 的子问题，因此有
T(n)=2T(n−1)+O(1)T(n) = 2T(n-1) + O(1)T(n)=2T(n−1)+O(1)
通过递归展开，可得时间复杂度为：
T(n)=O(2n)T(n) = O(2^n)T(n)=O(2n)
